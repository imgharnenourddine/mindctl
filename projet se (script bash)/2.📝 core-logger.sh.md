## 🎯 Ce que fait ce fichier

Contient la fonction `log_message()` utilisée par **tous les agents**. Elle écrit simultanément dans le terminal ET dans `/var/log/mindctl/history.log`.

Format imposé par le prof :

```
yyyy-mm-dd-hh-mm-ss : username : TYPE : message
```

---

## 💻 Le code

```bash
#!/bin/bash

source ~/mindctl/mindctl.conf

mkdir -p "$MINDCTL_LOG_DIR"

function log_message() {

    local TYPE="$1"
    local MESSAGE="$2"

    local DATE=$(date +"%Y-%m-%d-%H-%M-%S")
    local USER=$(whoami)
    local LIGNE="$DATE : $USER : $TYPE : $MESSAGE"

    echo "$LIGNE" | tee -a "$MINDCTL_LOG_DIR/history.log"
}
```

---

## 🔍 Explication ligne par ligne

`#!/bin/bash` → dit à Linux d'utiliser Bash pour exécuter ce script

`source ~/mindctl/mindctl.conf` → charge les variables de config (`MINDCTL_LOG_DIR`...) en mémoire

`mkdir -p "$MINDCTL_LOG_DIR"` → crée le dossier de logs s'il n'existe pas encore. `-p` évite une erreur si déjà existant

`function log_message()` → déclare la fonction. Elle ne s'exécute pas ici, elle est juste définie

`local TYPE="$1"` → `$1` = premier paramètre passé à la fonction. `local` = variable visible uniquement dans la fonction

`local MESSAGE="$2"` → `$2` = deuxième paramètre passé à la fonction

`local DATE=$(date +"%Y-%m-%d-%H-%M-%S")` → `$()` exécute une commande et met le résultat dans la variable. `date` retourne la date au format demandé

`local USER=$(whoami)` → `whoami` retourne le nom de l'utilisateur connecté

`local LIGNE="$DATE : $USER : $TYPE : $MESSAGE"` → assemble tous les morceaux en une seule ligne

`echo "$LIGNE" | tee -a "$MINDCTL_LOG_DIR/history.log"` → `tee` écrit à deux endroits simultanément : terminal + fichier. `-a` = ajouter à la fin sans écraser

## 🧪 Test

```bash
source ~/mindctl/core/logger.sh
log_message "INFOS" "test de démarrage mindctl"
log_message "ERROR" "test d'erreur"
cat /var/log/mindctl/history.log
```

`source` → charge le fichier dans le terminal pour rendre `log_message()` disponible

`log_message "INFOS" "..."` → appelle la fonction avec `$1=INFOS` et `$2=message`

`cat` → affiche le contenu du fichier pour vérifier que les lignes ont bien été écrites
