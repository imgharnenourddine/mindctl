## 🎯 Ce que fait ce fichier

Contient la fonction `appeler_llm()` utilisée par **tous les agents qui ont besoin d'intelligence**. Elle envoie un prompt à l'API Mistral AI et retourne la réponse en texte.

> [!info] Utilisée par : `depguard`, `analyzer`, `insight`

---

## 💻 Le code

```bash
#!/bin/bash

source ~/mindctl/mindctl.conf
source ~/mindctl/core/logger.sh
source ~/mindctl/core/errors.sh

function appeler_llm() {

    local PROMPT="$1"

    if [ -z "$MISTRAL_API_KEY" ]; then
        erreur 105
    fi

    local REPONSE=$(curl -s \
        -X POST "https://api.mistral.ai/v1/chat/completions" \
        -H "Authorization: Bearer $MISTRAL_API_KEY" \
        -H "Content-Type: application/json" \
        -d "{
            \"model\": \"$MINDCTL_MODEL\",
            \"messages\": [
                {
                    \"role\": \"user\",
                    \"content\": \"$PROMPT\"
                }
            ]
        }")

    if [ -z "$REPONSE" ]; then
        log_message "ERROR" "API Mistral — aucune réponse reçue"
        erreur 105
    fi

    local TEXTE=$(echo "$REPONSE" | jq -r '.choices[0].message.content')

    log_message "INFOS" "API Mistral — réponse reçue"

    echo "$TEXTE"
}
```

## 🔍 Explication ligne par ligne

`source ~/mindctl/mindctl.conf` → charge `MISTRAL_API_KEY` et `MINDCTL_MODEL` en mémoire

`source ~/mindctl/core/logger.sh` → charge `log_message()` pour loguer les appels API

`source ~/mindctl/core/errors.sh` → charge `erreur()` pour gérer les échecs

`local PROMPT="$1"` → `$1` = le texte envoyé à Mistral. Ex : `appeler_llm "analyse ce fichier"` → `$1` vaut `"analyse ce fichier"`

`if [ -z "$MISTRAL_API_KEY" ]` → `-z` vérifie si la variable est vide. Si la clé n'est pas configurée → erreur 105

`curl -s` → envoie une requête HTTP. `-s` = silent, cache la barre de progression

`-X POST` → type de requête POST — utilisé pour envoyer des données à une API

`-H "Authorization: Bearer $MISTRAL_API_KEY"` → envoie la clé API dans l'entête. `Bearer` = mot standard pour un token d'authentification

`-H "Content-Type: application/json"` → dit à l'API que les données envoyées sont en JSON

`-d "{...}"` → data — le corps de la requête contenant le modèle et le prompt

`\"model\": \"$MINDCTL_MODEL\"` → spécifie quel modèle Mistral utiliser. `\"` = guillemets échappés car on est dans une chaîne

`\"role\": \"user\"` → dit à Mistral que ce message vient de l'utilisateur

`\"content\": \"$PROMPT\"` → le texte qu'on envoie à Mistral

`if [ -z "$REPONSE" ]` → si `curl` n'a rien retourné → pas de connexion ou URL incorrecte → erreur 105

`jq -r '.choices[0].message.content'` → extrait uniquement le texte de la réponse JSON

```
{
  "choices": [
    {                          ← [0] premier élément
      "message": {
        "content": "Bonjour"   ← c'est ça qu'on veut
      }
    }
  ]
}
```

`-r` = raw → retourne le texte sans guillemets autour

`log_message "INFOS" "API Mistral — réponse reçue"` → logue que l'appel a réussi

`echo "$TEXTE"` → retourne le texte pour que l'agent qui a appelé la fonction puisse l'utiliser

## ⚠️ Problème rencontré

La variable `$LLM_URL` dans `mindctl.conf` était mal lue — caractère invisible ou espace.

**Solution appliquée :** remplacer `$LLM_URL` par l'URL directe dans le code :

```bash
-X POST "https://api.mistral.ai/v1/chat/completions" \
```
