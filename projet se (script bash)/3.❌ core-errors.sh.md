## 🎯 Ce que fait ce fichier

Contient deux fonctions :

- `afficher_aide()` → affiche la documentation du programme
- `erreur()` → affiche le message d'erreur + aide + arrête le script

> [!info] Appelée automatiquement après chaque erreur dans tous les agents.

---

## 💻 Le code

```bash
#!/bin/bash

source ~/mindctl/core/logger.sh

function afficher_aide() {
    echo ""
    echo "UTILISATION : mindctl [options] -p <paramètre>"
    echo ""
    echo "OPTIONS :"
    echo "  -h              Affiche cette aide"
    echo "  -f              Exécution via fork"
    echo "  -t              Exécution via threads"
    echo "  -s              Exécution via subshell"
    echo "  -l <dossier>    Dossier des logs"
    echo "  -r              Réinitialise la config (admin)"
    echo "  -a <agent>      Choisit l'agent"
    echo "  -p <chemin>     Fichier à analyser (OBLIGATOIRE)"
    echo "  -w              Mode pipeline"
    echo ""
    echo "AGENTS DISPONIBLES :"
    echo "  depguard | cleaner | analyzer"
    echo "  transformer | validator | insight"
    echo ""
    echo "CODES D'ERREUR :"
    echo "  100 → Option inexistante"
    echo "  101 → Paramètre -p manquant"
    echo "  102 → Fichier introuvable"
    echo "  103 → Permission refusée"
    echo "  104 → Format non supporté"
    echo "  105 → API inaccessible"
    echo "  106 → Agent inconnu"
    echo "  107 → Échec du traitement"
    echo ""
}

function erreur() {
    local CODE="$1"
    local MESSAGE=""

    case $CODE in
        100) MESSAGE="Option inexistante" ;;
        101) MESSAGE="Paramètre obligatoire manquant : -p <fichier>" ;;
        102) MESSAGE="Fichier ou dossier introuvable" ;;
        103) MESSAGE="Permission refusée — droits administrateur requis" ;;
        104) MESSAGE="Format de fichier non supporté" ;;
        105) MESSAGE="API Mistral inaccessible — vérifiez votre clé API" ;;
        106) MESSAGE="Agent inconnu — utilisez -h pour voir les agents disponibles" ;;
        107) MESSAGE="Échec du traitement" ;;
        *)   MESSAGE="Erreur inconnue" ;;
    esac

    log_message "ERROR" "Code $CODE — $MESSAGE"
    echo ""
    echo "[ERREUR $CODE] $MESSAGE"
    echo ""
    afficher_aide
    exit $CODE
}
```

---

## 🔍 Explication ligne par ligne

`source ~/mindctl/core/logger.sh` → charge `logger.sh` pour avoir accès à `log_message()`. Sans ça le script plante.

`function afficher_aide()` → déclare la fonction qui affiche la documentation. Appelée automatiquement après chaque erreur.

`echo ""` → affiche une ligne vide pour aérer l'affichage. `echo "texte"` affiche du texte dans le terminal.

`function erreur()` → reçoit un code d'erreur et fait trois choses : loguer + afficher le message + afficher l'aide.

`local CODE="$1"` → `$1` = le code passé en paramètre. Ex : `erreur 101` → `$1` vaut `101`.

`local MESSAGE=""` → variable vide pour l'instant — sera remplie par le `case`.

`case $CODE in` → compare `$CODE` à chaque valeur et exécute la ligne correspondante. Équivalent d'un `if/elif` mais plus lisible.

`100) MESSAGE="..." ;;` → si `$CODE` vaut `100` → assigner ce message. `;;` marque la fin de chaque cas — obligatoire.

`*) MESSAGE="Erreur inconnue" ;;` → cas par défaut si aucun code ne correspond.

`esac` → fin du `case`. C'est `case` écrit à l'envers.

`log_message "ERROR" "Code $CODE — $MESSAGE"` → écrit l'erreur dans `history.log` via `logger.sh`.

`echo "[ERREUR $CODE] $MESSAGE"` → affiche le message d'erreur dans le terminal.

`afficher_aide` → appelle la fonction du dessus pour montrer la doc automatiquement.

`exit $CODE` → arrête le script. `0` = succès, tout autre nombre = erreur.

---

## ⚠️ Pourquoi WSL se ferme

`exit $CODE` ferme toute la session WSL quand vous testez directement dans le terminal car vous êtes dans le shell principal.

---

## 🧪 Test — sans fermer WSL

```bash
bash -c 'source ~/mindctl/core/errors.sh && erreur 101'
bash -c 'source ~/mindctl/core/errors.sh && erreur 105'
```

`bash -c` → ouvre un Bash temporaire. Quand `exit` est appelé il ferme ce Bash temporaire seulement — pas votre WSL.
