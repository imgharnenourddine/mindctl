mindctl est un orchestrateur DevOps/données sous Linux.
Il produit ces fichiers après exécution :

1. CLEANER produit :
   /tmp/mindctl_{TABLE}_clean.csv
   Exemple : /tmp/mindctl_users_clean.csv
   Contenu : CSV nettoyé (doublons supprimés, nulls remplacés)
   Colonnes : id, nom, email, age, salaire

2. ANALYZER produit :
   /var/log/mindctl/reports/last_data.txt
   Contenu : statistiques de chaque table + interprétation LLM
   Format texte brut avec des lignes comme :
   "Colonne age : min=22.00 max=45.00 moy=30.50 | nulls=3"
   "Colonne email : plus_fréquent=gmail.com | uniques=8 | nulls=2"

3. DEPGUARD produit :
   /var/log/mindctl/reports/last_depguard.txt
   Contenu : conflits détectés dans les fichiers de config
   Format texte brut avec des lignes comme :
   "NIVEAU: CRITIQUE"
   "DESCRIPTION: version node:14 dans Dockerfile mais 18 dans package.json"
   "FICHIER: Dockerfile"
   "LIGNE_AVANT: FROM node:14"
   "LIGNE_APRES: FROM node:18"

Créer un serveur FastAPI avec CES ROUTES EXACTES :

━━━ ROUTE 1 : GET /api/csv/{table} ━━━
→ Lit le fichier /tmp/mindctl_{table}_clean.csv
→ Parse le CSV ligne par ligne
→ Retourne JSON :
{
  "table": "users",
  "total_lignes": 35,
  "colonnes": ["id","nom","email","age","salaire"],
  "donnees": [
    {"id":"1","nom":"Ali","email":"ali@gmail.com","age":"25","salaire":"3000"},
    ...
  ]
}
→ Si fichier n'existe pas → retourne erreur 404 avec message clair

━━━ ROUTE 2 : GET /api/stats ━━━
→ Lit /var/log/mindctl/reports/last_data.txt
→ Retourne le contenu brut dans JSON :
{
  "stats": "contenu complet du fichier last_data.txt"
}
→ Si fichier n'existe pas → retourne {"stats": "Aucune analyse disponible"}

━━━ ROUTE 3 : GET /api/depguard ━━━
→ Lit /var/log/mindctl/reports/last_depguard.txt
→ Parse le contenu et retourne JSON :
{
  "conflits": [
    {
      "niveau": "CRITIQUE",
      "description": "version node:14 dans Dockerfile mais 18 dans package.json",
      "fichier": "Dockerfile",
      "ligne_avant": "FROM node:14",
      "ligne_apres": "FROM node:18"
    }
  ],
  "total": 1
}
→ Si fichier n'existe pas → retourne {"conflits": [], "total": 0}

━━━ ROUTE 4 : GET /api/tables ━━━
→ Liste tous les fichiers /tmp/mindctl_*_clean.csv
→ Extrait les noms des tables
→ Retourne JSON :
{
  "tables": ["users", "clients", "produits"]
}

━━━ ROUTE 5 : GET / ━━━
→ Sert le fichier static/index.html

━━━ CONFIGURATION FASTAPI ━━━
→ Monter le dossier static/ pour servir index.html
→ Ajouter CORS pour permettre les requêtes JS :
  allow_origins=["*"]
  allow_methods=["*"]
  allow_headers=["*"]

━━━ SECTION 2 : Statistiques (analyzer) ━━━

━━━ SECTION 3 : Conflits DevOps (depguard) ━━━

→ Quand cliqué : appelle GET /api/depguard

   → NIVEAU en rouge si CRITIQUE, orange si WARNING
   → DESCRIPTION du conflit
   → FICHIER concerné
   → LIGNE_AVANT en rouge
   → LIGNE_APRES en vert

━━━ STYLE CSS ━━━
→ Fond sombre (#1a1a2e)
→ Texte blanc
→ Tableau avec bordures propres
→ Boutons bleus (#0066cc)
→ Cartes conflits avec bordure rouge/orange selon niveau
→ Titre principal : "mindctl — Dashboard de monitoring"
